HTMLは、Webページの構造を定義するマークアップ言語であり、ブラウザに文字や表などを表示させるために使用される。

一方、CSSはWebページのデザインを定義するものであり、HTMLで記述されたタグやレイアウトを装飾するためのものとなる。

したがって、ブログを作る上でHTMLとCSSを編集することは必須となる。

「Blogger」はGoogleが提供する無料のブログ作成サービスだが、BloggerにおけるHTML / CSSの関係性は少し独特なものとなっている。

この記事では、BloggerにおけるHTML / CSSの構成を整理し、カスタマイズの幅を広げるための備忘録としたい。

<h3 id="BloggerのHTMLCSS構成">BloggerのHTML/CSS構成</h3>

Bloggerは、以下の図のように「テーマのHTML」と「記事のHTML」とに分かれており、主にテーマのHTMLの中でCSSを記述する（デザインを定義する）形となっている。

CSSはテーマのHTMLの中で直接記述しても良いが、Blogger特有の機能「CSSを追加」で追加しても良い。

あるいは外部のCSSファイルとリンクしても良いが、Bloggerでは上の方法がメジャーかと思う。

<h4 id="テーマのHTMLテンプレート">テーマのHTML（テンプレート）</h4>

ブログ全体のレイアウトや構造を定義するものである。

ここでは、<b:section> や <b:widget> など、Blogger独自のタグが使われている。

このHTMLにCSSやJavaScriptを組み込むことで、ブログ全体のデザインと動作を決めることができる。

<h4 id="テーマのCSS">テーマのCSS</h4>

テーマのHTML内に <b:skin>〜</b:skin>タグで囲われた領域にCSSのコードを記述できる。

Bloggerの「CSSを追加」機能で追加されたCSSも、この中に組み込まれる。

ここで追加したCSSは、ブログ全体に適用される。

またテーマのCSSに後述するクラスを追加することで、個別記事の特定の部分のデザインも変更することができる。

<h4 id="記事のHTML投稿内容">記事のHTML（投稿内容）</h4>

各記事の本文部分であり、投稿エディターでHTMLビューにすることで確認できる。

<p>, <img>, <h1> などの通常のHTMLタグは使えるが、そのデザインは「テーマのCSS」で決められる。

例えば、テーマのHTML内で p { color: red; } と設定していれば、記事内の <p> も赤くなる。

記事毎に特定部分のデザインを変更したい場合は、こちらを編集することとなる。

<h3 id="ブログデザインの変更方法">ブログデザインの変更方法</h3>

ブログ全体のデザインを一括して変えるのか、それとも記事毎にデザインを変えるのかによって、多少方法は異なる。

<h4 id="全体のデザインを変えたい場合">全体のデザインを変えたい場合</h4>

ブログ全体のデザインを変える場合は以下の２通りあるが、Bloggerを使うのであれば最初の方法を推奨する。

<h5 id="テーマのHTMLの中でCSSを記述する方法">テーマのHTMLの中でCSSを記述する方法</h5>

１つ目のやり方は、テーマのHTMLの中にCSSを記述 / 編集するという方法である。


* テーマのHTML内に <b:skin>〜</b:skin>タグで囲われた領域にCSSのコードを記述する
* Bloggerの[「CSSを追加」機能](https://ip-note.blogspot.com/2025/05/bloggercss.html)でCSSのコードを追加する



CSSを追加するのであれば２の方法が便利だが、一度追加したCSSを編集する場合は１の方法で行う必要がある。

<h5 id="外部CSSファイルを読み込む方法">外部CSSファイルを読み込む方法</h5>

２つ目は、外部CSSファイルを読み込むという方法である。

具体的には、あらかじめサーバー内に自分で用意したCSSファイルを格納しておき、テーマのHTMLの<head>タグの中に、以下のような<link>タグを追加するという手順となる。


```
<pre class="prettyprint linenums"><link rel="stylesheet" type="text/css" href="CSSファイルのリンク先のURL">
```



外部CSSファイルを使う場合は、自分でCSSファイルをホスティングできるサーバーが必要となる。

Blogger自体はCSSファイルのアップロード機能を持っていないため、例えばGitHub Pagesといったような外部サービスを使うことになる。

ただし、外部サービスによっては商用利用を制限している場合もあるので、ブログにアフィリエイト広告を載せようとしている場合は問題ないか確認したほうが良さそうである。

<h4 id="記事ごとにデザインを変えたい場合">記事ごとにデザインを変えたい場合</h4>

あらかじめテーマのCSSにクラスを追加しておき、記事のHTML内でそのクラスを呼び出すことで、設定しておいたデザインを使うことができる。 

以下、赤枠を付ける簡単なコードの例で手順を説明する。

<h4 id="テーマのHTMLのCSSにクラスを追加">テーマのHTMLのCSSにクラスを追加</h4>

まず、CSSに以下のようなクラスを追加する。


```css
/* 赤枠を付けるクラス */
/* テーマのHTMLの<b:skin>内に書く */
.red-box{
  border: 2px solid red;
  padding: 10px;
  background-color: #ffecec;
}

```



<h5 id="記事のHTMLにクラス属性を指定">記事のHTMLにクラス属性を指定</h5>

次に記事へと移動し、投稿エディターでHTMLビューに切り替えた後、以下のようにクラス属性を指定したコードを挿入する。


```html
<div class='red-box'>
  これは赤枠のボックスです。
</div>

```



これで記事の投稿をプレビューすると、「これは赤枠のボックスです。」のテキストに赤枠が装飾されていることを確認できるかと思う。

このように、頻繁に記事の一部で使うデザインはクラスを追加しておくと使い勝手が良くなる。

なお、記事のHTML内に<style>タグを入れてデザインすることも可能だが、基本的に投稿本文内の<style>タグや<link>タグの使用はセキュリティ上制限されている。

たとえ書けたとしてもBloggerが自動的に除去する場合があるので、お勧めしない。
